# CryptoScan Burst Bot

Фоновый бот, который проверяет всплески объёма на Binance Futures и присылает алерты в Telegram — независимо от того, открыт ли у тебя браузер со скринером.

Логика полностью повторяет режим **"Объём в $"** из CryptoScan: ищет монеты, у которых за последние 30 минут хотя бы 10 свечей дали объём ≥ $1M, и при этом исключает "тяжеловесов" (BTC, ETH и подобные), для которых такой объём — это норма, а не всплеск интереса.

## Что понадобится

- Аккаунт на [Render.com](https://render.com) (бесплатный, без карты)
- Аккаунт на [GitHub](https://github.com) (бесплатный) — Render разворачивает код из git-репозитория
- Токен Telegram-бота и Chat ID — те же самые, что ты уже использовала в самом скринере

## Шаг 1 — выложить код на GitHub

1. Создай новый репозиторий на GitHub (можно приватный) — например `burst-bot`.
2. Залей туда файлы `burst_bot.py`, `requirements.txt` и этот `README.md` (через веб-интерфейс GitHub можно просто загрузить файлы кнопкой "Add file → Upload files", без терминала).

## Шаг 2 — создать Web Service на Render

⚠️ **Важно про регион сервера.** Binance официально блокирует запросы из США (и нескольких других стран) — это их собственная политика, а не баг. Render по умолчанию предлагает американский регион (Oregon/US-West или Ohio/US-East), и если оставить дефолт, бот будет получать ошибку `451 Client Error` при каждой попытке получить данные с Binance. **Обязательно выбери регион Frankfurt (Europe) или Singapore** — они не входят в список заблокированных Binance стран.

1. Зайди на [render.com](https://render.com), зарегистрируйся (можно через GitHub-аккаунт — это сразу даст доступ к репозиториям).
2. Нажми **New → Web Service**.
3. Выбери свой репозиторий `burst-bot`.
4. Заполни поля:
   - **Name**: любое (например `cryptoscan-burst-bot`)
   - **Region**: **Frankfurt (EU Central)** или **Singapore** — НЕ Oregon/Ohio
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python burst_bot.py`
   - **Instance Type**: **Free**
5. В разделе **Environment Variables** добавь две переменные:
   - `TG_BOT_TOKEN` → твой токен бота (тот же, что в настройках скринера)
   - `TG_CHAT_ID` → твой Chat ID (тот же, что в настройках скринера)
6. Нажми **Create Web Service**. Render соберёт и запустит бота — это займёт пару минут, прогресс виден в логах на странице сервиса.

После деплоя Render даст тебе публичный URL вида `https://cryptoscan-burst-bot.onrender.com` — открой его в браузере, должно показать что-то вроде:

```
CryptoScan Burst Bot работает.
Telegram: настроен
Последний цикл: Проверено 150 монет, активных всплесков: 2 (H, AERO)
```

Если вместо этого видишь в статусе `Ошибка получения тикеров: 451 Client Error` — значит сервис всё-таки запущен в американском регионе. Зайди в настройки сервиса на Render (Settings) и проверь/смени регион — если сменить регион существующего сервиса нельзя (зависит от текущего интерфейса Render), проще удалить сервис и создать заново, сразу выбрав Frankfurt или Singapore на шаге создания.

## Шаг 3 — не дать боту "заснуть"

Бесплатный план Render усыпляет Web Service после 15 минут без входящих запросов — а это значит фоновая проверка всплесков тоже остановится. Чтобы держать его всегда живым, нужен внешний сервис, который будет "пинговать" твой URL каждые несколько минут.

1. Зарегистрируйся на [UptimeRobot](https://uptimerobot.com) (бесплатно, без карты).
2. Создай новый монитор: **Add New Monitor**.
3. Тип — **HTTP(s)**, URL — тот, что дал тебе Render (например `https://cryptoscan-burst-bot.onrender.com`).
4. Интервал проверки — **5 минут** (бесплатный план UptimeRobot это поддерживает).
5. Сохрани.

Теперь UptimeRobot будет заходить на твой бот каждые 5 минут, не давая ему уснуть — а значит фоновый цикл проверки всплесков будет работать постоянно.

## Настройки бота

Все параметры находятся в начале файла `burst_bot.py` — их можно поменять перед деплоем, если хочешь другие условия:

- `BURST_WINDOW_MIN = 30` — окно поиска всплеска, в минутах
- `BURST_MIN_CANDLES = 10` — сколько свечей в этом окне должны превысить порог
- `BURST_THRESHOLD_USD = 1_000_000` — порог объёма на одну свечу
- `HEAVYWEIGHT_RATIO = 0.7` — если 70%+ свечей в широком 60-минутном окне "горячие", монета считается тяжеловесом и исключается
- `CHECK_INTERVAL_SEC = 60` — как часто повторять проверку (раз в минуту)
- `ALERT_COOLDOWN_SEC = 1800` — не слать повторный алерт по той же монете чаще раза в 30 минут

После изменения параметров просто запушь обновлённый файл в свой GitHub-репозиторий — Render автоматически пересоберёт и перезапустит сервис.

## Ограничения бесплатного варианта

- Первый запрос после "сна" может занять до минуты — пинг от UptimeRobot должен предотвращать сам сон, но если он всё же случится (например при сбое UptimeRobot), бот проснётся при следующем визите.
- Бесплатный план Render даёт ограниченное количество часов в месяц на все бесплатные сервисы суммарно — для одного небольшого бота этого обычно достаточно, но если планируешь несколько таких сервисов, может понадобиться платный тариф.
- Условия бесплatных тарифов у облачных провайдеров периодически меняются — если что-то в процессе выше не совпадает с текущим интерфейсом Render, ищи в их официальной документации актуальный способ задать переменные окружения и команду запуска для Python-проекта.
